--- TITLE ---
A user value named `sort` with its own `.sorted(a, b)` method is NOT hijacked by the sort.sorted binding intercept: the intercept is gated on the receiver resolving to the predeclared `sort` module (sema symbol), so this emits an ordinary Go method call, no tideSorted helper, no `import "sort"`. The field access `sort.id` exports to `sort.Id` even though `sort` names a package — the export gate is the sema symbol, not the spelling (name-match footgun).

--- INPUT ---
import fmt

class Sorter {
  let id: int
  sorted(a: int, b: int): int { return a + b }
}

func main() {
  let sort = Sorter{ id: 1 }
  fmt.println(sort.sorted(3, 4))
  fmt.println(sort.id)
}

--- GO ---
package main

import "fmt"

type Sorter struct {
	Id int `json:"id"`
}

func (t *Sorter) sorted(a int, b int) int {
	return a + b
}
func main() {
	var sort = &Sorter{Id: 1}
	fmt.Println(sort.sorted(3, 4))
	fmt.Println(sort.Id)
}

--- STDOUT ---
7
1

--- EXIT ---
0
